# Human Responder Workflow

This guide describes the standard workflow for support agents who handle customer conversations in Scale Flow.

## Agent lifecycle

1. Sign in and set status to **Available**.
2. Accept assigned conversation from queue/inbox.
3. Review customer profile, channel history, and tags.
4. Respond, clarify, and resolve the issue.
5. Mark outcome and close or escalate.

## Inbox handling model

- **Assigned to me**: active conversations owned by the current agent.
- **Team queue**: unassigned conversations waiting for pickup.
- **Escalated**: conversations requiring supervisor or specialist support.

## Response standards

- Send first response within target SLA.
- Confirm understanding before proposing a solution.
- Use approved macros/templates for consistency.
- Add internal notes for important context before transfer.

## Collaboration rules

1. Use internal mention comments when requesting help.
2. Reassign with a clear reason and next action.
3. Avoid duplicate replies by checking typing/lock status.
4. Inform customers when ownership changes.

## Escalation criteria

- Policy exceptions requiring approval.
- High-risk complaints or legal-sensitive requests.
- Multi-system incidents beyond frontline scope.
- Requests blocked by external dependencies.

## Best practices

- Keep messages short, clear, and action-oriented.
- Tag root cause and resolution type for analytics.
- Close conversations only after customer confirmation or clear closure conditions.
